Persistent, global key/value data store. Open collections, write data with put. Then retrieve data with get.

Keys and values are stored in range chunks. Chunks migrate to the region they're most frequently accessed from.

src/fly/cache/index.ts

bridge
bridge: any
Type : any

An API for accessing a regional, volatile cache. Data stored in @fly/cache can have an associated per-key time to live (TTL), and we will evict key data automatically after the elapsed TTL. We will also evict unused data when we need to reclaim space.

 * import cache from "@fly/cache"
 *
 * await cache.set("test-key", "test-value")
 *
 * const s = await cache.getString("test-key")
 *

See fly/cache/response for caching HTTP Response objects.

See fly/cache/global for global cache del/purge
cache
cache: object
Type : object
Default value : { get, getString, getMulti, getMultiString, set, expire, del, setTags, purgeTag, global }
